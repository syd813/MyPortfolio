AWS Automated Data Pipeline

Capstone Project Dec 2025 – Feb 2026 Millions of records per batch

Automated Cloud-Based Data Pipeline

A fully automated ETL pipeline built on AWS that ingests, validates, cleans, and warehouses large-scale recurring datasets — delivering analysis-ready data to ML, BI, and analytics teams with built-in data quality enforcement and zero manual intervention.

System Architecture

The Problem

Monthly NYC TLC datasets contain millions of records with missing values, inconsistencies, invalid timestamps, and fare calculation errors. Manual cleaning is slow, inconsistent, and does not scale. Existing ETL pipelines do not integrate data quality enforcement directly into processing, leading to unreliable downstream data.

No visibility for stakeholders and the team who wait for the data.

The Solution

Designed and implemented a fully automated cloud-native pipeline that embeds validation, cleaning, and imputation directly into the ETL workflow — ensuring only high-quality, structured data reaches downstream systems.

  • Automated ingestion with DynamoDB state tracking (no duplicates or missed runs)
  • Rule-based validation engine enforcing business constraints
  • Imputation layer to recover invalid records instead of discarding them
  • Data segmentation into valid, imputed, and bad datasets
  • Snowflake integration for analytics-ready storage
  • Monitoring UI for real-time pipeline visibility and stakeholder access

Pipeline Workflow

  • Extract monthly dataset → store in S3 (raw)
  • Validate records using business rules
  • Impute recoverable records
  • Store outputs (clean / imputed / bad)
  • Load clean data into Snowflake
  • Log metrics to DynamoDB → displayed in UI
